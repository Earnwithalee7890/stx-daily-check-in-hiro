# Guía de Contribución

¡Gracias por tu interés en contribuir a Stacks Builder Hub! Esta es una comunidad dedicada a construir herramientas premium para el ecosistema de Stacks.

## Cómo Empezar

1. **Bifurca el repositorio**: Crea tu propia copia del proyecto.
2. **Crea una rama**: `git checkout -b feature/mi-nueva-caracteristica`.
3. **Realiza tus cambios**: Asegúrate de seguir nuestros estándares de código.
4. **Prueba tus cambios**: Ejecuta `npm run test` para validar los contratos.
5. **Envía un Pull Request**: Describe tus cambios detalladamente.

## Estándares de Código

### Contratos Inteligentes (Clarity)
- Usa nombres descriptivos para funciones y variables.
- Siempre implementa `print` para eventos importantes.
- Asegúrate de que todos los errores estén definidos como constantes al principio del contrato.

### Frontend (React/Next.js)
- Prioriza la estética premium y el uso de glassmorphism.
- Usa `Vanilla CSS` para los estilos personalizados.
- Asegúrate de que los componentes sean responsivos.

## Reporte de Errores

Si encuentras un error, por favor abre un 'Issue' en GitHub con:
- Una descripción clara del problema.
- Pasos para reproducirlo.
- Comportamiento esperado vs. Comportamiento actual.

## Seguridad

Para vulnerabilidades de seguridad, por favor consulta nuestro [SECURITY.md](../SECURITY.md) y no abras un issue público.
